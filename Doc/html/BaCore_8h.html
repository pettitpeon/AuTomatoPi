TBaBoolRC BaCoreDestroyThread ( TBaCoreThreadHdl  hdl,
uint32_t  timeoutMs 
)

Thread destructor.

It signals the thread to end and waits a maximum of timeOutMs ms. If it timeouts, the thread detaches, stays alive, and releases the resources when the thread routine returns.

Returns
Error if bad handle or timeout and timeOutMs != 0, otherwise, success
Parameters
[in]hdlThread handle
[in]timeoutMsTimeout to wait in ms. If zero, no timeout and behaves like an asynchronous call
